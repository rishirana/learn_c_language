Qualifier

Qualifier give one of two properties to an identifier.

  • const - it declare object to be non-modifiable
  • volatile- it declare an item whose value can legitimately be changed by something beyond the control of the program in which it appears.

How to Use

  • The const keyword can be used to modify any fundamental or aggregate type,or a pointer to an object of any type , or a typedef If an item is declared with only the const type qualifier, its type is taken to be const int . A const variable ccan be initialized or can be placed in a read-only region of storage. The const keyword is useful for declaring pointers to const since this requires the function not to change the pointer in any way.

  • The compiler assumes that, at any point in the program , a volatile variable can be accessed by an unknown process that uses or modifies its value. Therefore regardless of the optimizations specified on the command line, the code for each assignment to or reference of a volatile variable must be generated even if it appears to have no effect.

  • An item can be both const and volatile , in which case the item could not be legitimately modified by its own program, but could be modified by some asynchronous process.
